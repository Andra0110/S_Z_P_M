# Requirements Analysis

## Project Goal

The goal of the project is to create a console-based clinic management system that allows management of patients, doctors and appointments.

---

# Functional Requirements

## Patient Management

The system shall allow:

- adding a patient
- removing a patient
- searching for a patient
- displaying all patients

---

## Doctor Management

The system shall allow:

- adding a doctor
- assigning specialization
- displaying doctors

---

## Appointment Management

The system shall allow:

- creating appointments
- cancelling appointments
- assigning patients to doctors
- displaying appointment schedules

---

## Data Persistence

The system shall allow:

- saving data to files
- loading data from files

---

# Non-Functional Requirements

- application written in C++
- console-based interface
- object-oriented design
- modular architecture
- use of Git and GitHub
- use of Doxygen documentation

---

# System Users

## Receptionist

Can:

- manage patients
- manage doctors
- schedule appointments

---

# Constraints

- no graphical interface
- local file storage
- single-user application
